2021中科大ctf大赛小记

发表于2021-11-09|更新于2021-11-12|网安学习记录
|阅读量:

前言

金秋十月，来南航的第三周。在室友的介绍下得知了中科大今年的ctf比赛。本着a过签到题就是赢的“决心”参加了这次比赛。最终“eng”做出了2/5的题目。下面结合自己的做题经历和官方放出的题解做一下小结。

签到

签到题目的内容类似于一个带时间的日历，初始时间为1970.1.1 00:00，每点击next一次会增加一秒，因为比赛是2021.10.23 12:00开始就顺理成章的想到应该把时间加至开赛时间，之后的工作就是计算两个时间点相隔的总秒数。百度一个日历计算器会大大方便计算。

进制十六——参上

题目图中将flag串打码，左侧给出了字符串的十六进制编码，将其手动输入十六进制ascii码转换器即可。

去吧！追寻自由的电波

是一道音频题，利用国际无线电通话拼写字母表代表字母进行朗读。同时音频进行了加速。因为自己参加ctf比赛经历极少，没有合适的音频题目处理软件。所以利用电脑上已有的播放器进行慢速播放（效果极差，仅速度变慢声调已久很高），甚至还为了一个降调插件下载了酷狗，拜这些无效工具所赐，最终还是完整的听出了flag是什么，记录提交，答案错误……翻来覆去听这段糊糊的音频觉得flag就是没问题。最后直接搁置了这道题。

直到某一时刻，心血来潮查找ctf音频题的一些wp，到了大佬们使用的软件Audacity：

https://audacity.onl/

下载，拖入题目，慢放3倍，眼泪哇哇的从眼睛倒流去耳朵（垃圾酷狗），最后发现因为原来不专业工具混淆了读音（PAPA听成了ALPHA）。这道题教育意义远大于题解本身，工欲善其事,必先利其器。

猫咪问答 Pro Max

听说猫咪问答是中科大每年的特色题目，搜索引擎用起来。

  • 信息安全俱乐部的章程的通过日期：这个俱乐部域名（sec.ustc.edu.cn）已经无法访问，因为我实在是没找到，也没有想到其他简单方法，就想着直接burp爆破，找到date字段设置一下跟题目答案一样的日期格式：

YYYYMMDD

​ 设置近几年的日期进行爆破即可（纯笨办法）。

​ 之后阅读到大佬们的题解学习到了其他方法，获取已经下线的网站的内容可以去 Web Archive 等相关网站去 找。最后可以找到原页面https://web.archive.org/web/20181004003308/http://sec.ustc.edu.cn/doku.php/codes。

中国科学技术大学学生信息安全俱乐部（以下简称“信息安全俱乐部”）是由中国科学技术大学对信息安全感兴趣、学习和研究信息安全的本科生、研究生自发组建的，由信息科学技术学院网络信息安全系（筹）提供专业指导的，开展信息安全领域相关活动的学生社团。（详见信息安全俱乐部社团章程)

本章程在 2015 年 5 月 4 日，经会员代表大会审议通过。

  • 俱乐部被评为五星级学生社团次数：中科大LINUX俱乐部首页有写。
  • 活动室牌子上的小字：LUG 的西区图书馆新活动室启用新闻页面 中有相关图片。
  • SIGBOVIK 2021 会报 的第 213 页中有。
  • 相关文件为： 愚人节的 RFC8962，找到举报相关的字段即可拿到答案

卖瓜

题目设定为：瓜有6元一斤和9元一斤的，最终要称出20斤的瓜。

题目并没有提供源代码只能手动尝试。起初对题目页面使用burp抓包，尝试修改两种西瓜的斤数，发现字段自动过滤非数字符号且传入浮点数会自动转换为整型。

最终考虑题目考察方向为大整数溢出，PHP 在运算结果超出 int 范围时，会返回 float 类型，进一步使用函数 intval() 来把 float 转为 int 时，如果这个 float 超出了 int 范围），结果是未定义的。尝试传入一个极大的参数：

1500000000000000000 * 9 // int(-4946744073709551616), 模 3 余 2

得到一个很小的负数，接下来就可以逐步尝试加入一个没有第一次那么大的参数，逐步把它加回到20即可，例如，一步到位的参数如下：

549638230412172404 * 9 // int(20)

透明的文件

题目描述为：

一个透明的文件，用于在终端中展示一个五颜六色的 flag。

可能是在 cmd.exe 等劣质终端中被长期使用的原因，这个文件失去了一些重要成分，变成了一堆乱码，也不会再显示出 flag 了。

之后下载题目文件，打开之后很懵，是大规模的形如下面的字符串：

[0;0H[20;58H[8;34H[13;27H[4;2H[38;2;1;204;177m [39m[14;10H[20;51H[23;4H[12;2H[38;2;2;207;173m [39m[19;61H[9;12H[22;8H[20;2H[38;2;3;210;169m [39m[3;23H[8;68H[19;10H[4;3H[38;2;4;214;165m [39m[19;23H[17;34H[11;52H[22;70H[12;3H[38;2;5;217;161m [39m[24;22H[2;25H[19;76H[19;3H[38;2;6;220;157m

因为毫无思路，这道题搁置到结束的倒数第二天，仔细扣题重点提示为：终端、五颜六色，便去搜索相关的资料，最终搜到一篇早年的博客：

https://blog.csdn.net/zhoujiaxq/article/details/22785803

这篇博客中介绍利用ANSI.SYS转义语言，在cmd命令行中取得意想不到的显示效果。根据语法只需要在每个[前加上ESC，之后再加载到命令行即可。起初加载flag会显示为大段字符中的空格，因为cmd多为黑色背景，可以全部选中字符或者替换掉所有的空格即可显示flag。下图为最终的效果（不得不说全部选中的那一刻有玩刮刮乐刮出奖金的惊喜感）。

旅行照片

社工题目，搜索蓝色KFC，可以得知，这是全国仅有一家的蒂夫尼蓝KFC，很幸运地在第一页就找到了 秦皇岛的蒂夫尼蓝色 KFC。也就是 这一家。对照地图，可以从视角和地图的对应关系得知拍照方向，从阴影位置得知拍照时间，而对于楼层就纯靠枚举了。最后在地图链接里可以找到电话信息。不过地图上的 KFC 标偏了，实际上正确的位置应当在 这个地方，可以在街景视图上看到建筑上的字。

flag助力大礼包

题目描述为需要用不同网段的ip地址点击flag助力地址，助力到1个flag时即可拿到flag。通过尝试和题目提示发现会对源地址进行检查， 位于同一个 /8 网络内的用户只能助力一次。而IPv4至多有256个/8网段。推断最终攒齐全部 256 个 /8 网段的助力才可获得全部 Flag。

而这里需要我们去伪造IP地址，题目服务器存在 X-Forwarded-For 欺骗漏洞。这一头标是不被认证且很容易被伪造的，因此可用构造出任意远程地址。

具体可见这里wp的第一题：

https://titaniumlja.github.io/2021/10/17/%E6%94%BB%E9%98%B2%E4%B8%96%E7%95%8C-Web/

X-Forwarded-For 的格式非常简单，只需要将源地址的 IP 地址记录其中即可。当存在多个中转服务器的时候，后续中转服务器会将自己的地址写在该头标的末尾，并使用逗号进行分割。因为对python使用不太熟练，所以我选择用burp抓包利用爆破功能的方式做题。首先打开助力地址，点击助力并抓包，send it to intruder。用脚本生成一个0-255.X.X.X的一个IP地址字典。在请求的头文件中加入 X-Forwarded-For字段，用爆破选择标签标记这个字段和助力者IP两个字段，之后将生成的IP字典导入burp，start sttack即可。这里需要注意的是把burp默认选择的符号转义关掉，不然.会被转成编码形式。

或者：

另外，记录一下做题时查到的一些关于c程序运行时内存分布的资料：

  • 代码段 text 存放函数指令
  • 数据段 data 存放已初始化的全局变量和静态变量，
  • 只读数据段 rodata 存放只读常量或const关键字标识的全局变量
  • bss段 bss 存放未初始化的全局变量和静态变量，这些变量由于为初始化，所以没有必要在elf中为其分配空间。bss段的长度总为0。
  • 调试信息段 debug 存放调试信息
  • 行号表 line 存放编译器代码行号和指令的duiing关系
  • 字符串表 strtab 存储elf中存储的各种字符串
  • 符号表 symtab elf中到处的符号，用于链接 注：局部变量在函数中运行中动态的分配和释放，在elf中并没有对应的段。使用malloc和free函数动态申请的空间在运行时在堆中分配，elf中没有对应的段。
